# `@sunflower0305/claude-proxy` v1.1.0

Second public npm release of `claude-proxy`, published as `@sunflower0305/claude-proxy`.

## Highlights

- runtime proxy state is now isolated per `createApp()` instance for safer multi-app embedding
- provider and model inference were refactored to support dynamic provider identification more cleanly
- local integration coverage now exercises `qwen`, `glm`, and `minimax`
- GitHub Actions CI now runs build, local integration tests, and coverage reporting
- Kimi defaults and related docs were updated to `kimi-k2.6`
- startup output and internal logging were cleaned up for a quieter CLI experience

## Installation

```bash
npx @sunflower0305/claude-proxy
```

or:

```bash
npm install -g @sunflower0305/claude-proxy
claude-proxy
```

## Quick Start

```bash
export PROVIDER=qwen
export QWEN_API_KEY=your-qwen-api-key

export ANTHROPIC_BASE_URL=http://localhost:8080
export ANTHROPIC_API_KEY=any-string-works

claude-proxy
```

## Notes

- requires Node.js `>=18`
- the proxy listens on `http://localhost:8080` by default
- clients should point `ANTHROPIC_BASE_URL` at the local proxy and can use any non-empty `ANTHROPIC_API_KEY`
