# `@sunflower0305/claude-proxy` v1.1.2

Patch release of `claude-proxy`, published as `@sunflower0305/claude-proxy`.

## Highlights

- default provider changed from `qwen` to `deepseek`
- default DeepSeek model changed from `deepseek-chat` to `deepseek-v4-pro`
- README now includes a provider defaults table for all supported providers
- README maintenance-only release verification and CI publishing sections were removed from the npm-facing package documentation

## Installation

```bash
npx @sunflower0305/claude-proxy
```

or:

```bash
npm install -g @sunflower0305/claude-proxy
claude-proxy
```

## Quick Start

```bash
export PROVIDER=deepseek
export DEEPSEEK_API_KEY=your-deepseek-api-key
export DEEPSEEK_MODEL=deepseek-v4-pro

export ANTHROPIC_BASE_URL=http://localhost:8080
export ANTHROPIC_API_KEY=any-string-works

claude-proxy
```

If `PROVIDER` is unset or invalid, the proxy now starts with `deepseek`.

- requires Node.js `>=18`
- the proxy listens on `http://localhost:8080` by default
- clients should point `ANTHROPIC_BASE_URL` at the local proxy and can use any non-empty `ANTHROPIC_API_KEY`
